# BoatScrubCalendar Growth & AI Discovery Plan

This plan is designed to help https://boatscrubcalendar.com/ rank for high-intent searches, appear in AI-generated answers, and convert traffic into paid subscriptions.

## 1) Positioning: own one clear category

Primary category to own:
- **"Boat scrubbing tide calendar UK"**

Primary value proposition:
- "Find the best tide windows to scrub your boat in minutes, with UK tide predictions and a simple monthly planner."

Use this exact framing in:
- Homepage H1 and intro paragraph
- Title/meta description
- Open Graph/Twitter text
- Product descriptions and directory submissions

## 2) SEO foundation (week 1)

1. Keep one canonical domain and use it everywhere.
   - Ensure every social/share tag references `https://boatscrubcalendar.com/`.
2. Ship AI-readable website docs.
   - Publish `/llms.txt` for model crawlers with purpose, audience, and key pages.
3. Expand structured data beyond app schema.
   - Add `WebSite` + `SearchAction`.
   - Add `FAQPage` schema for the most common user questions.
4. Strengthen core metadata by page intent.
   - Homepage: broad "boat scrubbing + UK tides" terms.
   - Blog pages: one long-tail phrase each (e.g. "best tide height for boat scrubbing").

## 3) Content flywheel for rankings (weeks 2-6)

Create 2 practical articles per month targeting "how/when" intent:

- "Best Tide Height for Boat Scrubbing in UK Harbours"
- "How to Plan Scrubbing Around Spring and Neap Tides"
- "Boat Scrubbing Checklist Before You Dry Out"
- "Admiralty Tide Data vs Free Sources: What Changes for Scrubbing?"
- "Top 10 UK Marinas for Easy Scrubbing Access"

Content format requirements:
- Answer-first intro (40-60 words)
- Step-by-step sections
- One worked example (real UK location)
- FAQ block at the end
- Internal links to subscription/product CTA

## 4) AI-answer optimization (AEO)

To improve visibility in Google AI Overviews and chat assistants:

1. Write short answer blocks under each H2:
   - "In short: ..." with a direct factual answer.
2. Keep facts explicit and attributable:
   - Mention data source type (Admiralty, harmonic prediction, etc.).
3. Add concise "entity" references:
   - UK marinas, ports, and tidal terms as dedicated sections.
4. Build quoteable snippets:
   - 1-2 sentence definitions for terms like "drying window" and "safe scrub window".

## 5) Promotion channels that match buyer intent

### High intent (priority)
- UK boating forums and yacht club newsletters
- Marina Facebook groups (local geography + tide need)
- Partnerships with marine service providers (cleaning/maintenance)

### Scalable acquisition
- YouTube shorts/reels: "This weekend's scrub windows in [location]"
- Email digest: weekly "best scrub windows" by region
- Directory listings: Product Hunt alternatives, UK marine directories, niche boating tools pages

## 6) Conversion improvements

- Add "free preview" without signup for one location/week.
- Add proof section:
  - testimonials from real boat owners
  - number of locations covered
  - date of last tide data refresh
- Add CTA near top:
  - "See next safe scrub window"

## 7) 90-day KPI targets

- Organic impressions: +100%
- Non-branded clicks: +60%
- Top-10 keywords: 15+ long-tail terms
- Free-to-paid conversion: >4%
- Referral traffic from boating communities: +30%

## 8) Operating cadence

Weekly:
- Publish or update one indexable page
- Refresh internal links from homepage/blog
- Share one practical tip post in a community channel

Monthly:
- Review Search Console query gains/losses
- Expand pages that are ranking #8-#20
- Add 3 new FAQ entries from real support questions
